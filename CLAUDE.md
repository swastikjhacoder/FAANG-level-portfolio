# CLAUDE.md

## Purpose
Guidelines for AI-assisted development and collaboration.

---

## 1. Development Philosophy
- Modular design
- Clean architecture
- Security-first
- Scalable systems

---

## 2. Coding Guidelines
- Follow existing patterns
- Avoid unnecessary paradigms
- Prioritize readability

---

## 3. Architecture Rules
- Route → Middleware → Controller → UseCase → Domain → Repository → DB
- Never skip layers

---

## 4. Security Rules
- Sanitize inputs
- Validate DTOs
- Enforce authorization
- Use secure cookies

---

## 5. Collaboration Rules
- Small, meaningful commits
- Document major changes
- Maintain backward compatibility

---

## 6. Anti-Patterns to Avoid
- Fat controllers
- Direct DB access from routes
- Mixing business logic with infrastructure

---

## 7. Goal
Maintain consistency and high engineering standards across the project.
